GELU

Gaussian Error Linear Unit. Eine Aktivierungsfunktion, also das „krumme Ding" das zwischen den linearen Schichten sitzt damit der Stapel mächtiger wird als eine einzelne Schicht.

Was es macht

Grob gesagt: positive Werte lässt es fast unverändert durch. Negative Werte werden Richtung null gedrückt, aber nicht hart auf null gesetzt wie bei ReLU, sondern weich, mit einer leichten Kurve.

\text{GELU}(x) = x \cdot \Phi(x)

\Phi(x) ist die kumulative Normalverteilung. Im Grunde eine Wahrscheinlichkeit dafür wie weit links oder rechts der Wert liegt. Positiv und groß? \Phi ist fast 1, der Wert bleibt. Negativ? \Phi geht gegen 0, der Wert verschwindet.

Warum nicht ReLU?

ReLU macht einfach max(0, x). Alles unter null wird null, alles drüber bleibt. Funktioniert, aber der harte Knick bei null ist mathematisch unschön und kann dazu führen dass Neuronen komplett absterben (immer null ausgeben, nie wieder lernen).

GELU hat einen weichen Übergang. Praktisch performt es in Transformer-Modellen leicht besser. GPT-2, GPT-3, BERT nutzen alle GELU.

In PyTorch

import torch.nn as nn

activation = nn.GELU()
output = activation(torch.tensor([-1.0, 0.0, 1.0, 2.0]))
# tensor([-0.1587, 0.0000, 0.8413, 1.9545])

Verwandte

  • ReLU: max(0, x) – der Klassiker, einfach und schnell
  • SiLU / Swish: x · sigmoid(x) – ähnlich wie GELU, wird in LLaMA genutzt
  • SwiGLU: Kombination aus Swish und einem gated linear unit, aktueller Standard in vielen LLMs

Alle machen im Grunde dasselbe: einen nichtlinearen Knick einbauen damit das Netz mehr kann als nur lineare Abbildungen.
