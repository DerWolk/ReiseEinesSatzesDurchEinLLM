Autoregressive Generierung

So erzeugen LLMs Text: ein Token nach dem anderen, wobei jedes neue Token auf allen bisherigen basiert. Dann wird es angehängt und der ganze Apparat läuft nochmal.

Der Loop

while not done:
    logits     = model(input_ids)[:, -1, :]
    probs      = F.softmax(logits / temperature, dim=-1)
    next_token = torch.multinomial(probs, num_samples=1)
    input_ids  = torch.cat([input_ids, next_token], dim=-1)
    done       = (next_token.item() == eos_id)
  1. Alle bisherigen Token durchs Modell jagen
  2. Am Ende fällt eine Verteilung über mögliche nächste Token raus
  3. Eins wird gewürfelt (Sampling) oder das wahrscheinlichste genommen (Greedy)
  4. Anhängen, von vorne

Warum autoREgressiv?

„Regressiv" heißt rückwärts bezogen. Jede Vorhersage bezieht sich auf das was vorher kam. Wie bei einer Zeitreihe wo der nächste Wert von den bisherigen abhängt.

Warum nicht alles auf einmal?

Weil jedes Token von allen vorherigen abhängt. Du kannst Token 5 nicht vorhersagen ohne Token 1–4 zu kennen. Und Token 6 hängt von 1–5 ab, inklusive dem gerade erst generierten Token 5.

Bei der Eingabe geht das parallel (die sind ja alle schon da). Aber bei der Ausgabe zwangsläufig sequentiell.

Deshalb ist es langsam

Ein Absatz mit 200 Token = 200 komplette Durchläufe durch alle 80 Transformer-Blöcke. Jeder Durchlauf produziert nur EIN Token. Deshalb merkst du bei ChatGPT dass die Antwort Stück für Stück kommt.

Deshalb wird in Token abgerechnet

Ein Output-Token = ein Durchlauf durchs ganze Modell. Das ist teuer. Token sind nicht nur die Einheit von Text. Token sind die Einheit von Rechenaufwand.

KV-Cache

Optimierung: Statt bei jedem neuen Token alle bisherigen Token nochmal komplett durch Attention zu schicken, speichert man die K- und V-Vektoren der vorherigen Token. Spart massiv Rechenzeit, kostet GPU-Speicher. Tradeoff.
