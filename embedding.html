Embedding

Ein Embedding ist ein Vektor, also eine Reihe von Zahlen, der ein Token im hochdimensionalen Raum positioniert. Tokens die in ähnlichen Kontexten auftauchen landen nah beieinander.

Die Idee

Du kannst Wörter nicht direkt verrechnen. Aber Zahlen schon. Also gibst du jedem Token eine Position in einem Raum mit tausenden Dimensionen. In diesem Raum bedeutet Nähe so etwas wie Ähnlichkeit.

„Python" sitzt nah bei „JavaScript". Weit weg von „Bratwurst". Nicht weil das jemand programmiert hat, sondern weil das Modell beim Training gelernt hat dass „Python" und „JavaScript" in ähnlichen Sätzen auftauchen.

Technisch

Eine Lookup-Tabelle. So viele Zeilen wie Token im Vokabular (100.000–200.000), jede Zeile ein Vektor mit typischerweise 4.096 Werten. Du gibst eine Token-ID rein, du kriegst den passenden Vektor raus.

import torch.nn as nn

emb = nn.Embedding(num_embeddings=150_000, embedding_dim=4096)
vector = emb(torch.tensor([15490]))  # Vektor für "Python"

Der berühmte Befund

Mikolov 2013, Word2Vec:

\vec{\text{König}} - \vec{\text{Mann}} + \vec{\text{Frau}} \approx \vec{\text{Königin}}

Vektorarithmetik funktioniert mit Bedeutungen. Das ergibt sich rein aus Statistik, niemand hat Bedeutung einprogrammiert.

Haken

Das Embedding ist kontextfrei. „Bank" hat immer den gleichen Startvektor, egal ob Parkbank oder Sparkasse. Erst Attention löst das auf.
