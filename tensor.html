Tensor

In PyTorch das Wort für ein mehrdimensionales Zahlengitter. Nicht mehr, nicht weniger. Alles was durchs Modell fließt ist ein Tensor.

Dimensionen

  • 0D: Eine einzelne Zahl. torch.tensor(42) – ein Skalar.
  • 1D: Eine Reihe von Zahlen. torch.tensor([1, 2, 3]) – ein Vektor.
  • 2D: Ein Gitter. Shape [8, 4096] – acht Vektoren mit je 4.096 Werten. Das ist was nach dem Embedding rauskommt.
  • 3D: Ein Stapel von Gittern. Shape [batch, seq_len, dim] – mehrere Sätze auf einmal.
  • 4D+: Kommt bei Bildern vor (batch, channels, height, width). Für Text selten nötig.

Shape

Die wichtigste Info an einem Tensor. Sagt dir wie viele Dimensionen es gibt und wie groß jede ist.

import torch

x = torch.randn(8, 4096)
print(x.shape)  # torch.Size([8, 4096])

Wenn du einen Fehler hast ist es meistens ein Shape-Mismatch. Irgendwo passen die Dimensionen nicht zusammen.

Auf der GPU

Tensoren können auf der CPU oder GPU leben. Auf der GPU rechnet alles parallel und ist drastisch schneller.

x = x.to("cuda")  # ab auf die GPU

Ist das wie ein NumPy Array?

Ja, fast identisch. Tensor ist PyTorchs Version davon, mit dem Bonus dass Gradienten automatisch berechnet werden (Autograd) und dass es auf GPUs läuft. Du kannst jederzeit hin und her konvertieren: tensor.numpy() und torch.from_numpy(array).
